# 中国采购服务网站使用说明

本说明面向运营人员及管理员，帮助快速搭建、配置与维护网站。

## 1. 环境准备
- 安装 Node.js 18+。
- 进入项目目录 `web/` 后执行 `npm install` 安装依赖。
- 复制 `.env.example` 为 `.env.local`，按实际信息填写：
  - `ADMIN_PASSWORD`：后台登录密码。
  - `ADMIN_SESSION_TOKEN`：任意安全随机值。
  - `NEXT_PUBLIC_GA_ID`：GA4 ID（若暂未启用，可留空）。
  - `WHATSAPP_NUMBER`、`WHATSAPP_LINK` 等联系方式。

## 2. 本地运行与构建
- 开发模式：`npm run dev`，浏览器访问 `http://localhost:3000`。
- Lint 检查：`npm run lint`。
- 生产构建：`npm run build`，随后 `npm start` 可验证生产包。

## 3. 内容配置
网站大部分文案集中在：
- `src/data/companyProfile.ts`：公司信息、联系方式、图片区等。
- `src/data/siteContent.ts`：导航、Hero、Services、Pricing、案例、FAQ、Contact 等文本。
修改完成后重新运行 `npm run dev` 或刷新即可看到最新内容。

## 4. 联系表单与数据
- 前端提交到 `/api/contact`，数据保存在 `data/contact-submissions.json`。
- 目前为文件存储，正式上线建议接入数据库并配置邮件通知。
- 定期备份 `data/` 文件夹，确保询盘不会丢失。

## 5. 管理后台
- 访问 `/admin-login` 输入 `.env.local` 中设置的密码。
- 登录后进入 `/admin`，可查看、筛选、刷新、导出 CSV。
- 点击 “Logout” 退出账号。

## 6. 占位图片与二维码
- 所有图片位于 `public/images/`，目前为占位 SVG。
- 上线前替换为真实办公室、样品、仓库、微信二维码等照片，确保信任感。

## 7. 部署与安全建议
- 部署到支持 HTTPS 的平台（如 Vercel、Azure、阿里云等）。
- 设置环境变量、开启自动备份、配置 WAF/防火墙。
- 如需 reCAPTCHA 或更复杂的反垃圾方案，可在 `ContactForm` 和 `/api/contact` 中扩展验证逻辑。

## 8. 后续扩展
- 接入邮件服务（Nodemailer 等）自动通知。
- 将 `data/contact-submissions.json` 替换为数据库（Supabase、PlanetScale、MongoDB 等）。
- 在 Admin 中新增搜索、标签、状态管理等功能。

如有新的需求或改动，请更新本说明，确保团队成员共享最新流程。祝顺利！
